Local OS AI Assistant for Windows

Python-based AI assistant that executes PowerShell/CMD commands on Windows. Works with local LLMs (Ollama, LM Studio) or cloud APIs (OpenAI, Anthropic, etc.).

⚠️ Safety first – Every command requires your confirmation before execution.
Features

    🤖 OS automation – Describe what you want, AI generates the command

    🖥️ Direct execution – !ps <command> or !cmd <command>

    🔍 Auto web search – For factual questions (toggle with /toggle)

    📜 Command logging – Saves to command_log.txt

    🔄 Multi-step chaining – AI reads output and decides next steps

Requirements

    Windows with PowerShell/CMD

    Python 3.7+

bash

pip install requests duckduckgo-search

Setup

    Save script as os_assistant.py

    Choose provider:

        Ollama (port 11434) – needs model pulled

        LM Studio (port 1234) – needs loaded model

        API – OpenAI, Anthropic, or compatible endpoint

    Run: python os_assistant.py

Usage

Type what you want, e.g.: Show all text files in current folder

The AI proposes a command – you confirm before execution.

Commands:

    exit – quit

    /toggle – toggle web search

    /search <query> – force search

    !ps <command> – run PowerShell directly

    !cmd <command> – run CMD directly

⚠️ Recommended Models

For reliable OS automation, use large enough models (7B+ parameters) or cloud models. Small models (e.g., Phi-3 Mini, TinyLlama, <7B) typically lack the reasoning and command-generation accuracy needed for multi-step OS tasks.

Recommended:

    Cloud: GPT-4, Claude 3.5+, GPT-3.5-Turbo

    Local (7B+): Llama 3.x (8B+), Qwen 2.5 (7B+), Mistral (7B+)

    Local (larger): Llama 3 70B, CodeLlama 34B

Small models will produce incorrect commands or fail at chaining.
